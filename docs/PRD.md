# PRD: Automated UI Image Generation Pipeline

## Goal

Build a modular pipeline that accepts a base UI image plus a structured description of target elements, then identifies, extracts, restyles, recomposes, reviews, and exports a new UI image.

The MVP focuses on orchestration, file management, stage independence, and stable data contracts. Model integrations are added incrementally behind adapters.

## Users

- UI/UX designers who want fast style exploration from existing screenshots.
- Product teams that need consistent UI image variants.
- Engineers experimenting with VLM, segmentation, and generative image workflows.

## MVP Scope

The first version must:

- Provide a runnable CLI.
- Provide a local UI for non-technical operation.
- Accept a structured JSON job config.
- Create a predictable run directory.
- Execute each pipeline step independently.
- Persist every stage's inputs and outputs.
- Produce a final output artifact, even if generated by placeholder logic.
- Record enough metadata to debug and replay runs.

## Non-Goals For MVP

- Production-grade YOLO/VLM review integration.
- Production-grade inpainting or large-model style transfer.
- Batch queue system.
- Figma/PSD export.
- Human-in-the-loop annotation interface.

## Success Criteria

- A new contributor can run the sample config in one command.
- Every stage has a clear input and output JSON contract.
- Replacing one placeholder stage with a real model does not require changes to other stages.
- Run artifacts are easy to inspect manually.

## Open Decisions

- Preferred model stack for detection and segmentation.
- Whether final production output should be image-only, layered design data, or Figma-first.
- Whether style generation should be local, API-based, or hybrid.
- How much manual correction should be allowed between stages.
